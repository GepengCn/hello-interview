`synchronized` 是 Java 中的一个关键字，用于实现线程同步，确保多线程环境下对共享资源的安全访问。当一个方法或代码块被 `synchronized` 修饰时，同一时刻只有一个线程可以执行这段代码，从而防止多线程下的数据不一致和线程安全问题。

### 底层原理

`synchronized` 在底层依赖于 Java 对象头中的监视器锁（monitor lock）。当线程进入 `synchronized` 修饰的方法或代码块时，它会尝试获取锁。如果锁不可用，线程会被阻塞直到锁被释放。

### JDK 1.6之后的优化

在 JDK 1.6 之后，`synchronized` 经历了几项重要优化：

1. **轻量级锁**：当没有竞争出现时，减少锁的开销，通过在对象头中存储锁指针来实现。

2. **偏向锁**：优化单一线程访问的场景。当锁被同一个线程多次获取时，偏向锁可以减少不必要的锁竞争开销。

3. **锁膨胀和锁降级**：根据竞争情况，`synchronized` 锁可以在轻量级锁和重量级锁之间转换，并在适当的时候进行锁降级。

这些优化显著提高了 `synchronized` 在不同场景下的性能，尤其是在锁竞争不激烈的情况下。
